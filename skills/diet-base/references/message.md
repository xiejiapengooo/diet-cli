# +message

用于收集用户饮食信息，并规范输出以便插入数据库。

## 策略

- 一次最多问 1~2 个问题
- 优先补最关键信息
- 已提供信息不重复问
- 能推断就不问

`信息优先级`
1. 食物（必须）
2. 餐型（早餐/午餐/晚餐/零食）
3. 进食时间
4. 份量

`对话示例：`

用户：刚吃了个三明治<br>
助手：是当早餐还是午餐呢？

用户：早餐<br>
助手：大概几点吃的？（或默认当前时间）

---

## 自动推断规则

`时间 → Meal`
- 06:00–10:00 → 早餐
- 11:00–14:00 → 午餐
- 17:00–21:00 → 晚餐
- 其他 → 零食

`默认值`
- 时间缺失 → 当前时间（精确到小时）
- 份量缺失 → 标准份（1个/1碗/1杯/100g）：

`模糊描述处理`
- “吃了点” → 小份
- “很多” → 1.5~2倍
- “一份套餐” → 拆分食物结构

## 营养计算

`输出字段`
- Calories（kcal）
- Protein（g）
- Carbs（g）
- Fat（g）

`规则`
- 按食物拆分计算再汇总
- 使用常见数据库估算
- 允许误差 ±20%

## 食物规则

`格式`

food = 食物1（份量）
food = 食物2（份量）
food = 食物3（份量）

`示例`
- 三明治（1个）
- 咖啡（1杯）
- 米饭（1碗）
- 鸡胸肉（150g）

## 标题规则

title = {餐型} - {食物1 + 食物2 + 食物3}

`示例`
- 早餐 - 三明治 + 咖啡

## 时间规则

1. 先以用户当前时区解释时间词（若未给出时区，默认使用会话时区）
2. 再换算成绝对日期时间字符串，传给 `--at`、`--from`、`--to`
3. 执行后在回复中回显本次使用的绝对时间，避免歧义

`示例（当前日期：2026-04-02）`

- "今天早餐 8 点" -> "2026-04-02 08:00"
- "明天中午 12:30" -> "2026-04-03 12:30"
- "昨天晚上 7 点" -> "2026-04-01 19:00"

## 输出格式（最终）

- 标题（title）：早餐 - 三明治 + 咖啡
- 餐型（meal）：早餐
- 时间（eat_at）：2026-03-27 08:30
- 食物（food）：
  - 三明治（1个）
  - 咖啡（1杯）
- 热量（calories）：350 kcal
- 蛋白质（protein）：14 g
- 碳水（carbs）：40 g
- 脂肪（fat）：12 g
