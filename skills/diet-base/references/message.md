# +message

用于收集并标准化 `add` 命令所需参数。

## 目标字段

`add` 命令必须拿到以下字段：

1. `meal`：`breakfast|lunch|dinner|snack`
2. `foods`：必须使用 `食物1(份量) + 食物2(份量) + 食物3(份量)` 的拼接格式
3. `at`：绝对时间字符串（`YYYY-MM-DD HH:mm`）
4. `calories`：整数，`>= 0`，如果没提供，走营养值计算规则
5. `protein`：整数，`>= 0`，如果没提供，走营养值计算规则
6. `carbs`：整数，`>= 0`，如果没提供，走营养值计算规则
7. `fat`：整数，`>= 0`，如果没提供，走营养值计算规则

## foods 字段格式（必须）

- 使用 `食物名(份量)` 作为单项格式
- 多个食物用 ` + ` 连接
- 目标格式：`食物1(份量) + 食物2(份量) + 食物3(份量)`
- 食物数量可按实际增减，但每一项都必须带 `(...)` 份量

示例：

- `米饭(1 bowl) + 清蒸鱼(150g) + 西兰花(100g)`
- `酸奶(200ml) + 蓝莓(50g) + 坚果(20g)`

## 对话策略

- 一次最多问 1 到 2 个问题，优先补缺失必填项
- 已提供信息不重复问
- 能从上下文推断就不问
- 涉及“今天/昨天/明天”等相对时间，必须转换为绝对时间

## 自动推断规则

`时间 -> meal`

- `06:00-10:59` -> `breakfast`
- `11:00-14:59` -> `lunch`
- `17:00-21:59` -> `dinner`
- 其他 -> `snack`

`默认值`

- 若用户没给时间：使用会话当前时间并对齐到分钟
- 若用户没给份量：写成常见份量（如 `1 bowl` / `1 cup` / `100g`）


## 营养计算

`输出字段`
- calories（kcal）
- protein（g）
- carbs（g）
- fat（g）

`规则`
- 按食物拆分计算再汇总
- 使用常见数据库估算
- 允许误差 ±20%

## 时间规则

1. 必须优先使用用户已配置的时区解释时间词（如“今天/昨天/明天”）
2. 若用户尚未设置时区，先询问用户位置（至少国家+城市，必要时补充州/省）并通过 `user:timezone` 完成设置
3. 若用户未提供位置，不要使用会话/系统默认时区（如 `Asia/Shanghai`）进行推断或直接落库
4. 完成时区设置后，再生成绝对时间字符串给 `--at`
5. 执行后在回复中回显绝对时间，避免歧义

示例（当前日期：`2026-04-07`）：

- 今天早餐 8 点 -> `2026-04-07 08:00`
- 明天中午 12:30 -> `2026-04-08 12:30`
- 昨晚 7 点 -> `2026-04-06 19:00`

## 输出模板（供 +add 使用）

```text
meal: breakfast
foods: oatmeal(1 bowl) + banana(1) + milk(250ml)
at: 2026-04-07 08:00
calories: 420
protein: 18
carbs: 62
fat: 11
```

## 组装命令

```bash
diet add \
  --meal "<meal>" \
  --foods "<foods>" \
  --at "<at>" \
  --calories <calories> \
  --protein <protein> \
  --carbs <carbs> \
  --fat <fat>
```
